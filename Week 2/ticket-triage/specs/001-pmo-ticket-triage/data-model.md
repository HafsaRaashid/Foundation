# Data Model: PMO Ticket Triage Dashboard

**Feature**: `specs/001-pmo-ticket-triage`
**Date**: 2026-06-29

---

## Entity: Ticket

Represents a single inbound work request in the PMO queue.

### Fields

| Field | Type | Nullable | Default | Notes |
|-------|------|----------|---------|-------|
| `id` | `Int` | No | auto-increment | Primary key. Matches the ID from `tickets.json` seed fixture. |
| `title` | `String` | No | — | Human-readable description of the request. Max 255 chars (app-layer). |
| `priority` | `String?` | Yes | `null` | Constrained to `"P0"`, `"P1"`, `"P2"` at the API layer via Zod. `null` = untagged. SQLite has no native enum; constraint enforced by application. |
| `owner` | `String?` | Yes | `null` | Free-text person name. Max 100 chars (Zod). `null` = unassigned. |
| `status` | `String` | No | `"open"` | `"open"` or `"closed"`. Read-only in this release — no UI to change status. |
| `createdAt` | `DateTime` | No | `now()` | Used to order tickets within a priority group (oldest first). |

### Prisma Schema

```prisma
model Ticket {
  id        Int      @id @default(autoincrement())
  title     String
  priority  String?
  owner     String?
  status    String   @default("open")
  createdAt DateTime @default(now())
}
```

### Validation Rules

Applied at the API boundary via Zod (not in the database schema):

- `priority` MUST be one of `"P0"`, `"P1"`, `"P2"`, or `null` (to clear a tag).
- `owner` MUST be a string of ≤ 100 characters, or `null` (to clear an assignment).
- A `PATCH` request body MUST include at least one of `priority` or `owner`; an
  empty body is rejected with HTTP 400.

### State Transitions

```
Untagged (priority = null)
  └─► Tagged (priority = "P0" | "P1" | "P2")   via PATCH /tickets/:id
        └─► Re-tagged (different priority)       via PATCH /tickets/:id
        └─► Untagged (priority = null)           via PATCH /tickets/:id { priority: null }

Unowned (owner = null)
  └─► Owned (owner = "<name>")                  via PATCH /tickets/:id
        └─► Re-assigned (different owner)        via PATCH /tickets/:id
        └─► Unowned (owner = null)               via PATCH /tickets/:id { owner: null }
```

Status (`open`/`closed`) is set by the seed fixture and is read-only in this release.

### Dashboard Grouping Logic

Tickets are grouped on the server (in `app/page.tsx`) before rendering:

```
Groups (render order):
  P0  — tickets where priority = "P0"
  P1  — tickets where priority = "P1"
  P2  — tickets where priority = "P2"
  Untagged — tickets where priority IS NULL
```

Within each group, tickets are ordered by `createdAt ASC` (oldest first).
Only tickets where `status = "open"` are fetched.

## Seed Fixture Shape

`prisma/fixtures/tickets.json` is expected to be an array of objects matching:

```json
[
  {
    "id": 1,
    "title": "Example request title",
    "priority": null,
    "owner": null,
    "status": "open"
  }
]
```

Fields `priority`, `owner` are optional in the fixture (default to `null`).
Field `status` is optional (defaults to `"open"`).
Field `createdAt` is auto-generated by Prisma on seed insert.
